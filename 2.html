02. 자바스크립트 문법

console.log

코드상에서 에러가 발생하면 값을 알 수가 없어서 console.log를 사용해서 실행 결과를 중간중간 체크하면서 오류를 찾아갈 수 있습니다.

오류 검출용이라는 표현보다 값을 콘솔에 로그를 남겨서 눈으로 확인하기 위한 도구라고 이해하면 좋을것 같습니다.

    ex code)
    for(var i = 0; i < 5; i++) {
        console.log(i);
    }

주석

    //var plus = 1 + 2;

주석을 먼저 작성하고 주석을 기준으로 자바스크립트 코드를 작성합니다.
한글로 자기 생각을 정리할 수 있어야 자바스크립트 코드를 논리적으로 작성할 수 있습니다.

기본 문법

ES5 스펙에 '기본 문법'이라는 용어는 없으며 코드를 작성할 때 기본이 되는 문법을 모아서 다루기 위해 분류한 것입니다.

코드 작성 위치

    sports= '수영,축구';
    sports= '수영,' +
            '축구';

위의 두 작성법은 동일하게 작동하기에 사람이 보기 편하도록 작성하면 됩니다.

대소문자 구분

"ABC", "abc", "Abc" 는 각각 다르게 해석됩니다. 대부분 소문자로 시작하지만 대문자로 시작하는 것도 있습니다.

공백

    1. var sports = "swim";
    2. var sports="swim";
    3. varsports="swim";

1,2번은 동일하게 동작하지만 3번은 var의 변수 선언기능과 sport라는 단어의 구분이 없어져 비 정상적인 작동을 하게 됩니다.

토큰

토큰은 자바스크립트 코드를 구성하는 최소 단위입니다.
위에서 표현된 ( var, sports , =, swim ) 각각이 토큰입니다

문장 부호

    { } ( ) [ ] . ; , < > <= ...
    부호마다 기능이 드르므로 연산자 , 대괄호, 중괄호, 소괄호와 같이 기능 중심으로 세분하여 부릅니다.

세미콜론

    var soccer = 11 var basketball = 5 //오류
    =>
    var soccer = 11; var basketball = 5; //정상

자바스크립트는 세미콜론이 자동 삽이되어 편리하지만 세미콜론을 생략하는 습관은 바람직하지 않습니다.

유니코드

    console.log('\u0030');
    console.log('\u0039');
    console.log('\u0041');
    console.log('\u0061');
    console.log('\u1100');
    console.log('\uac00');

유니코드는 6자리로 작성하며 자바스크립트는 유니코드 3.0 버전에 정의된 문자를 사용합니다.

링크 : 유니코드 3.0 문서

화이트스페이스

Whitespace문자는 사람 눈에 보이지 않는 문자를 나타냅니다.
소스 텍스트(주석 포함)의 가독성을 높이기 위해 사용합니다. (수평탭, 수직탭, 폼 넘기기, 공백 등등.. -책 19p 참고)
유니코드를 사용전에 브라우저 지원 여부를 체크할 필요가 있습니다.

행 분리 문자

행 또는 문단 끝에서 행을 분리하는 문자를 의미합니다.
행 분리 문자 일람표 -책 20p 참고

이스케이프 시퀸스

Sequence의 사전적 의미는 "연달아 일어남, 연속, 순서, 차례"이며 이중에서 연속이 자바스크립트의 시퀸스의 의미에 가깝습니다.

Escape는 역슬래시(\)를 의미합니다.

Escape + Sequence = 이스케이프 시퀸스라고 하며 문자열로 해석되지 않고 특수한 기능을 수행합니다. -책 21p 참고

식별자 이름

변수, 함수 등을 식별하기 위한 이름을 의미하며 식별자 또는 이름으로 부르기도 합니다.
이름의 첫 문자는 영문자, 언더스코어(_), 달러마크($)를 사용하며 이후에 연속하여 숫자를 사용할 수 있습니다.

  • 함수와 변수 이름은 소문자로 작성합니다. soccer
  • 단어와 단어를 연결할 때 두 번째 단어의 첫 문자를 대문자로 작성합니다. soccerSports
  • 대문자 대신에 언더바를 사용할 수도 있습니다. soccer_sports
  • 오브젝트 이름의 첫 문자는 대문자로 작성합니다. Swim
  • 값이 변하지 않는 상수는 이름 전체를 대문자로 작성합니다. MAX_VALUE, PI
  • 이외에도 여러 가지가 있지만, 프로그램 개발에 앞서 명명 기준을 정하는 것이 바람직 합니다.

변수 선언

변수란 값이 변하는 영역을 의미하며 변수를 정의하는 것을 선언이라고 합니다.
선언하면 메모리에 설정되며 메모리에 접근하려면 이름이 있어야 하므로 변수는 이름을 사용하여 선언해야 합니다.

아래와 같이 var(variable), 하나이상의 공백, 변수 이름 순서로 작성하고 끝에 세미콜론을 작성합니다.

    var sports;
    var soccer;

    var sports, soccer;
    var sports,
        soccer;

변수 초깃값 할당

처음으로 변수에 할당하는 값을 초기값이라고 합니다.
초기값을 선언하지 않으면 undefined를 할당합니다.
변수를 선언하지 않고 변수를 사용하면 에러가 발생하지만 변수값이 undefined이면 변수를 선언한 것이므로 에러가 발생하지 않습니다.

    var sports = 'swim';
    var soccer=11;

    var sports = 'swim',
        soccer = 11;

변수 타입

숫자, 문자 와 같은 변수의 형태를 의미합니다.
정적인 언어에서는 숫자 타입이면 숫자만 할당, 문자 타입이면 문자만 할당할 수 있지만 동적인 언어인 자바스크립트는 변수 타입을 구분하지 않습니다.
하지만 데이터 타입(Data Type)은 구분합니다.
데이터 타입이란 변수에 설정된 값을 기준으로 값이 숫자면 숫자 데이터 타입이고 값이 문자면 문자 데이터 타입입니다.
변수에 값을 할당할때 비교하지 않으며 변수값을 사용하여 연산할때 데이터 타입을 구분합니다.

결정된 자바스크립트 특징

변수에 할당한 값을 기준으로 데이터 타입을 구분하는 것

변수 삭제 불가

변수에 설정된 값을 지울 수는 있어도 변수를 삭제할 수는 없습니다.

상수

값이 변하지 않는 영역을 의미합니다. 변수와 마찬가지로 상수에 접근하기 위해서는 상수를 식별할 수 있는 이름이 필요하며,
상수 선언 방법은 변수 선언 방법과 같으며 차이가 있다면 값이 변하지 않는다는 점입니다.
자바스크립트에서 제공하는 상수가 아닌 프로그램에서 정의한 상수는 값을 변경할 수 있으므로 의미적인 선언입니다.
상수를 영문 대문자로 선언하는것은 코딩 관례이며 값을 변경하지 말라는 암시입니다.

상수 사용 목적

항상 고정된 값을 사용하기 위해서입니다.

상수 종류

크게 두종류로 구분합니다.
자바스크립트 제공하는 상수(MAX_VALUE, MIN_VALUE와 같이 숫자로 표현할 수 있는 최대값 최소값을 상수로 제공합니다. 변경이 불가능하며 변경하면 에러가 발생합니다.)
개발자 소스코드로 선언한 상수("var ONE = 1" 형태와 같이 프로그램에서 선언한 상수는 값을 변경할 수 있습니다. 근본적으로 변수와 동일합니다.)
개발자가 부여한 상수는 사람이 인식하기 위한 의미적인 상수입니다.

키워드, 예약어

키워드

자바스크립트 기능을 가진 단어를 의마합니다. 따라서 프로그램에서 식별자 이름으로 사용할 수 없습니다. -책 27p 참고

예약어

현재 자바스크립트에서 사용하고 있지 않지만 향후 사용할 수 있으므로 예약해둔 단어를 의미합니다. 예약으롤 식별자 이름으로 사용하면 에러가 발생합니다. -책 28,29p 참고
